# Code of Conduct

ResearchQ is an open, collaborative project. We want everyone — contributors, issue reporters, reviewers, and users — to have a constructive experience participating in the project.

## Expectations

Participants in this project are expected to:

- Be respectful and professional in all project spaces (issues, pull requests, discussions, email, and any other communication channels).
- Assume good intent. When giving feedback on code or ideas, critique the work, not the person.
- Accept constructive feedback gracefully. If a review or comment is unclear, ask a follow-up question rather than escalating.
- Focus on what is best for the project and its users.
- Credit others for their ideas and contributions.

## Unacceptable behavior

The following are not acceptable in any project space:

- Personal attacks, insults, or derogatory remarks directed at other participants.
- Sustained disruption of discussions or review processes.
- Publishing another participant's private information (such as email address or physical location) without their explicit permission.
- Any conduct that a reasonable person would consider inappropriate in a professional setting.

## Scope

This Code applies in all project spaces and when an individual is representing the project in public. It applies to all participants equally, regardless of their role or level of contribution.

## Reporting

If you observe behavior that concerns you, please email **prangon.inovace@gmail.com**. Reports will be reviewed promptly and treated with discretion.

When you report, it helps to include:

1. Where the behavior occurred (issue URL, PR URL, etc.).
2. A short description of what happened.
3. Any context that may help in understanding the situation.

## Enforcement

The project maintainers are responsible for clarifying and enforcing this Code. Responses to a confirmed violation may include, at the maintainers' discretion:

- A private conversation with the participant.
- A public correction or retraction.
- Temporary or permanent removal of the participant's ability to participate in the project.

Maintainers who do not follow this Code in good faith may face consequences determined by the other maintainers.

## Acknowledgments

This Code is inspired by the spirit of the open-source community's shared norms around professional collaboration. It is deliberately short and direct so it can be read and understood in a few minutes.
